### Pre-existing Issues Found

1. **Read resource ownership gap:** `helix://clients/{id}/goals` and `helix://clients/{id}/sessions` use service role key (bypasses RLS) without application-level ownership verification. A coach who knows another coach's client UUID can read their data via these resources.
2. **Inconsistent join pattern:** `list_sessions` tool uses soft join + post-filter vs inner join used by resource equivalents.
3. **Duplicate protocol version:** `protocolVersion: "2024-11-05"` hardcoded at lines 2284 and 2454 in index.ts — Phase 19 must reconcile.

### Cross-Phase Wiring Risks

2. Phase 20 should resolve TOOL-04 (remove read tools) before TOOL-01/TOOL-02 to avoid translating/annotating tools that will be deleted
3. POL-01 and TOOL-02 both touch `executeTool()` error return paths — should be coordinated

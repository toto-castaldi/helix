# Milestone v1.5: Versioning GSD

**Status:** SHIPPED 2026-02-21
**Phases:** 16-17
**Total Plans:** 2

## Overview

Replace automatic date-time versioning with GSD milestone version, visible across all three apps (coach, live tablet, landing page) with GitHub repository link.

## Phases

### Phase 16: CI/CD Pipeline Cleanup

**Goal**: Build pipeline extracts version from GSD milestones instead of generating date-time stamps
**Depends on**: Nothing (first phase of v1.5)
**Plans**: 1 plan

Plans:
- [x] 16-01-PLAN.md — Replace date-time versioning with milestone extraction and clean up README

**Details:**
- Replaced `Generate version` step with `Extract milestone version` in deploy.yml
- Version extraction chain: PROJECT.md (active milestone) -> MILESTONES.md (latest shipped) -> "dev" fallback
- Removed `Update README`, `Commit version update`, `Push version commit` steps
- Removed `permissions: contents: write` and checkout token (no longer needed)
- Cleaned auto-generated version line from README.md

### Phase 17: Version Display

**Goal**: Users can see the current Helix version on every app surface and access the source repository
**Depends on**: Phase 16
**Plans**: 1 plan

Plans:
- [x] 17-01-PLAN.md — Add version display to live tablet and landing page with GitHub link

**Details:**
- Live tablet: version text in date-select header next to user email (subtle gray)
- Landing page: footer with version and clickable GitHub link (https://github.com/toto-castaldi/helix)
- Coach app: already had version display from previous work, format updated
- All three apps use consistent pattern: `import.meta.env.VITE_APP_VERSION || 'dev'`

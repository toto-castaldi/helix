const clientGoalsMatch = uri.match(/^helix:\/\/clients\/([^\/]+)\/goals$/)

**2d. Add resource handler for `helix://exercises/tags/{tag}` in `readResource()`:**

Add this handler right after the `helix://exercises/tags` handler block (after the `uniqueTags` return):

```typescript
// helix://exercises/tags/{tag}
const exercisesByTagMatch = uri.match(/^helix:\/\/exercises\/tags\/([^\/]+)$/)
if (exercisesByTagMatch) {
  const tag = decodeURIComponent(exercisesByTagMatch[1])
  const { data, error } = await supabase
    .from("exercises")
    .select(`
      id, name, description, lumio_card_id,
      exercise_tags!inner(tag)
    `)
    .eq("exercise_tags.tag", tag)
    .or(`user_id.eq.${userId},user_id.is.null`)
    .order("name")

  if (error) throw new Error(`[database_error] ${error.message}`)
  return [{ uri, mimeType: "application/json", text: JSON.stringify(data || [], null, 2) }]
}
```

The handler uses `exercise_tags!inner(tag)` join with `.eq("exercise_tags.tag", tag)` to filter exercises that have the specified tag. It uses `!inner` join so exercises without the tag are excluded. The `or` clause ensures only default exercises (user_id IS NULL) and the coach's custom exercises are returned.

**Important:** The `helix://exercises/tags` exact match must remain BEFORE the regex match for `helix://exercises/tags/{tag}`, otherwise `helix://exercises/tags` would match the regex with an empty capture. The current ordering in readResource (exact match first, then regex) is correct.

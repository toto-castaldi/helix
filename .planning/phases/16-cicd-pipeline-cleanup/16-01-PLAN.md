  <files>.github/workflows/deploy.yml</files>
  <action>
Modify `.github/workflows/deploy.yml` with these specific changes:

1. **Replace the "Generate version" step** (lines 22-26). The new step should:
   - First try to extract the active milestone from `.planning/PROJECT.md` by grepping for `Current Milestone: vX.Y` and extracting the `vX.Y` portion
   - If that fails (no match), fallback to extracting the highest `## vX.Y` from `.planning/MILESTONES.md` using `grep -oP '## \Kv\d+\.\d+' .planning/MILESTONES.md | sort -rV | head -1` (sort needed because entries aren't ordered newest-first)
   - If both fail, fallback to `dev`
   - Output the result as `version` (same output name so downstream steps work unchanged)
   - Example implementation:
     ```yaml
     - name: Extract milestone version
       id: version
       run: |
         VERSION=$(grep -oP 'Current Milestone: \Kv\d+\.\d+' .planning/PROJECT.md 2>/dev/null || true)
         if [ -z "$VERSION" ]; then
           VERSION=$(grep -oP '## \Kv\d+\.\d+' .planning/MILESTONES.md 2>/dev/null | sort -rV | head -1 || true)
         fi
         if [ -z "$VERSION" ]; then
           VERSION="dev"
         fi
         echo "version=$VERSION" >> $GITHUB_OUTPUT
         echo "Milestone version: $VERSION"
     ```

2. **Delete the "Update README with version" step entirely** (lines 28-34). Remove the entire step block.

3. **Delete the "Commit version update" step entirely** (lines 36-42). Remove the entire step block.

4. **Remove `permissions: contents: write`** (lines 8-9). This permission was only needed for the git push in the commit step. Without the commit step, the default read permission suffices. Remove both the `permissions:` key and the `contents: write` value.

5. **Remove the `token` parameter from the Checkout step** (line 19-20). The `token: ${{ secrets.GITHUB_TOKEN }}` was needed for push permissions. With no push, default checkout works fine. Keep the `uses: actions/checkout@v4` but remove the `with:` block containing `token:`.

6. **Keep all VITE_APP_VERSION references unchanged** in the three build steps (main, live, landing). They already reference `${{ steps.version.outputs.version }}` which will now output `v1.5` instead of `2026.02.19.1603`.

7. **Keep backup artifact naming unchanged** -- it already uses `${{ steps.version.outputs.version }}` which will now produce names like `db-backup-v1.5` instead of `db-backup-2026.02.19.1603`.

IMPORTANT: Do NOT modify any other steps in the workflow. The build steps, deploy steps, nginx steps, supabase steps, and edge function deploy steps must remain exactly as they are.

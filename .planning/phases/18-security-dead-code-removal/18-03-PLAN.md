<objective>
Add ownership verification to all unprotected write tools so a coach cannot modify another coach's data through the MCP server.

Purpose: SEC-02 requires that 6 write tools (update_session, delete_session, complete_session, update_session_exercise, remove_session_exercise, reorder_session_exercises) verify coach ownership before executing mutations. Research identified 2 additional unprotected tools (duplicate_session, add_session_exercise) with the same vulnerability. This plan creates shared ownership helper functions and applies the check-then-mutate pattern to all 8 unprotected tools.

Output: All write tools secured with ownership verification. Violations return "not found" and are logged as warnings.
</objective>

<task type="auto">
  <name>Task 1: Create shared ownership verification helpers</name>
  <files>supabase/functions/helix-mcp/index.ts</files>
  <action>
Add two shared ownership verification helper functions near the top of the file (after the Supabase client initialization, before the tool handlers):

1. **`verifySessionOwnership(supabase, userId, sessionId)`**:
   - Queries `sessions` with inner join on `clients` to check `user_id` matches
   - Returns `{ owned: true }` if the session exists AND belongs to the coach
   - Returns `{ owned: false }` if the session does not exist OR belongs to another coach
   - On `owned: false` when the session exists but belongs to another coach, log a warning:
     `console.warn(\`[SECURITY] Ownership violation: user=${userId} attempted to access session=${sessionId}\`)`
   - Use the exact Supabase pattern from research:
     ```typescript
     const { data, error } = await supabase
       .from("sessions")
       .select("id, client:clients!inner(user_id)")
       .eq("id", sessionId)
       .single()
     ```

2. **`verifySessionExerciseOwnership(supabase, userId, sessionExerciseId)`**:
   - Queries `session_exercises` with nested inner joins through `sessions` and `clients`
   - Returns `{ owned: true, sessionId: string }` if owned (sessionId useful for some tools)
   - Returns `{ owned: false }` if not found or belongs to another coach
   - On violation, log warning:
     `console.warn(\`[SECURITY] Ownership violation: user=${userId} attempted to access session_exercise=${sessionExerciseId}\`)`
   - Use the nested join pattern from research:
     ```typescript
     const { data, error } = await supabase
       .from("session_exercises")
       .select("id, session_id, session:sessions!inner(client:clients!inner(user_id))")
       .eq("id", sessionExerciseId)
       .single()
     ```

3. **`verifyClientOwnership(supabase, userId, clientId)`**:
   - Queries `clients` table directly: `.from("clients").select("id").eq("id", clientId).eq("user_id", userId).single()`
   - Returns `{ owned: true }` or `{ owned: false }`
   - On violation, log: `console.warn(\`[SECURITY] Ownership violation: user=${userId} attempted to access client=${clientId}\`)`
   - Needed for `duplicate_session`'s `new_client_id` parameter and `add_session_exercise`'s implicit session-through-client chain

Per user decision: "Implement a shared helper function used by all write tools for consistent behavior" and "Verify ownership through parent session chain: exercise -> session -> client -> user_id"

<task type="auto">
  <name>Task 2: Add ownership checks to all 8 unprotected write tools</name>
  <files>supabase/functions/helix-mcp/index.ts</files>
  <action>
Add ownership verification to each of the following 8 write tools using the check-then-mutate pattern. Per user decision: "Return 'not found' when a coach tries to access/modify another coach's resource -- do not reveal that the resource exists."

**Session-level tools (use `verifySessionOwnership`):**

1. **`update_session`**: Before updating, call `verifySessionOwnership(supabase, userId, session_id)`. If not owned, return `{ content: [{ type: "text", text: "Session not found" }] }`.

2. **`delete_session`**: Before deleting, call `verifySessionOwnership(supabase, userId, session_id)`. If not owned, return "Session not found".

3. **`complete_session`**: Before marking complete, call `verifySessionOwnership(supabase, userId, session_id)`. If not owned, return "Session not found".

4. **`duplicate_session`**: Before duplicating, call `verifySessionOwnership(supabase, userId, session_id)` for the source session. Additionally, if `new_client_id` is provided, call `verifyClientOwnership(supabase, userId, new_client_id)` to ensure the target client also belongs to this coach. If either fails, return "Session not found" or "Client not found" respectively.

5. **`add_session_exercise`**: Before adding an exercise, call `verifySessionOwnership(supabase, userId, session_id)`. If not owned, return "Session not found".

6. **`reorder_session_exercises`**: Before reordering, call `verifySessionOwnership(supabase, userId, session_id)`. If not owned, return "Session not found".

**Exercise-level tools (use `verifySessionExerciseOwnership`):**

7. **`update_session_exercise`**: Before updating, call `verifySessionExerciseOwnership(supabase, userId, session_exercise_id)`. If not owned, return `{ content: [{ type: "text", text: "Exercise not found" }] }`.

8. **`remove_session_exercise`**: Before removing, call `verifySessionExerciseOwnership(supabase, userId, session_exercise_id)`. If not owned, return "Exercise not found".

**Important implementation notes:**
- Per user decision: "Check-then-mutate pattern: first query verifies ownership, second query performs mutation"
- Per user decision: "Verify ownership on ALL operations including creates: before creating a session, verify client_id belongs to coach; before adding an exercise, verify session belongs to coach"
- The existing `create_session` and `create_training_plan` already have ownership checks -- do NOT modify them
- Response messages should be in English to align with Phase 20's planned English translation (use "Session not found", "Exercise not found", "Client not found")
- Each tool's existing mutation logic stays the same -- only add the ownership check BEFORE the mutation

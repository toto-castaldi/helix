<objective>
Create Edge Function that handles the full token update flow: save new PAT to database, push to Docora via PATCH API, and reset sync status.

Purpose: When a coach updates their GitHub PAT token to fix a sync failure, the system needs a backend endpoint that atomically: (1) saves the new token to the Helix database, (2) pushes the update to Docora so they use the new token for git operations, and (3) resets the sync status to 'pending' so Docora auto-retries the failed sync.

Output: New `docora-update-token` Edge Function, updated `docora.ts` shared helper with PATCH support, updated deploy workflow.

**2. Create supabase/functions/docora-update-token/index.ts:**

Follow the exact same patterns as `docora-register/index.ts` for auth, CORS, error handling, and Supabase client creation.

Request body interface:
```typescript
interface RequestBody {
  repositoryId: string
  newToken: string
}
```

Flow:
1. Handle CORS preflight (OPTIONS)
2. Reject non-POST methods (405)
3. Validate Authorization header
4. Create Supabase service role client
5. Validate JWT and get user
6. Parse request body -- validate repositoryId and newToken are present (400 if missing)
7. Verify user owns the repository (query lumio_repositories with eq id + eq user_id) -- 404 if not found
8. Check repository has docora_repository_id (400 if null -- can't update token if not registered with Docora)
9. Call Docora PATCH API to update token:
   ```typescript
   const docoraResult = await docoraApiCall(
     `/api/repositories/${repository.docora_repository_id}`,
     "PATCH",
     { github_token: newToken }
   )
   ```
10. If Docora call fails, return 502 with error message (do NOT save to DB if Docora rejects)
11. If Docora call succeeds, update lumio_repositories:
    ```typescript
    await supabase
      .from("lumio_repositories")
      .update({
        access_token: newToken,
        sync_status: "pending",
        sync_error_message: null,
        sync_failed_at: null,
      })
      .eq("id", repositoryId)
    ```
12. Return 200 with `{ success: true, message: "Token updated successfully" }`

Import from shared:
```typescript
import { docoraApiCall } from "../_shared/docora.ts"
```

Error responses follow same JSON format as docora-register: `{ error: "message" }`

IMPORTANT: Do NOT proceed with DB update if Docora PATCH fails -- the order is Docora first, then DB. If Docora rejects the token, the coach needs to know and try a different one.
